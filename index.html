Mercado Libre Internal tooling Data visualization

Dashboard
Tool

A dashboard giving logistics supervisors real-time visibility into package flow — replacing manual spreadsheets and reducing package loss by 8% in the first quarter.

Role
Senior UX Designer
Timeline
3 months
Users
Floor supervisors & team leads
Hero image — Dashboard overview

The problem

The return-to-station process had a visibility gap. Operators could log received packages, but had no way to see what happened next. KPIs were tracked in offline spreadsheets — updated manually, always out of date, and impossible to act on in real time.

When packages went missing, there was no way to trace where things had gone wrong. Loss went undetected, and decisions were made on stale data.

Understanding the flow

The hardest part wasn't the interface — it was understanding the process. Return-to-station involves multiple handoff points, edge cases, and exception states that aren't obvious until you spend time with the people running it.

We mapped the full flow and ran discovery sessions with supervisors and team leads to understand what information they actually needed to make decisions during a shift — and what was just noise.

Process flow — return-to-station mapping

Key design decisions

Testing revealed that exception states needed to be immediately scannable. When alerts were buried in tables, operators missed them under pressure. We moved toward a card-based layout for critical metrics, with clear visual weight for anything that needed action.

The goal was a dashboard you could read in under ten seconds — then drill down only when something needed investigation.

Wireframes — layout exploration
High-fidelity mockups — final design

Outcome

The tool shipped after 3 months and replaced offline tracking entirely. Supervisors gained a live view of package status and daily KPIs for the first time. The team ran minimal iterations post-launch — the core model held up well.

Results
  • 8% reduction in package loss in Q1 of use
  • Replaced manual spreadsheet tracking entirely
  • First measurable traceability for the return-to-station team

Mercado Libre Mobile Workflow design

Handshake

Replacing paper-based package handovers with a mobile-first digital flow — adopted by over 1,000 operators across the network.

Role
Senior UX Designer
Timeline
3 months
Scale
1,000+ operators
Hero image — Handshake mobile flow

The problem

Every package transfer between operators required both parties to sign a paper form. The system worked until it didn't — forms got lost, signatures were illegible, and when a package went missing there was no reliable record to trace. It created disputes instead of resolving them.

Designing for real conditions

This tool would be used by operators in physical locations — on mobile, under time pressure, with limited tolerance for friction. That context shaped every decision.

We tested several confirmation patterns. The key finding: any step that required manual input — typing, selecting, confirming with a tap — created drop-off. We stripped the flow down to scan or QR only. No manual input, no confirmation screens. The accountability record is created automatically at the moment of transfer.

Flow iteration — friction reduction

Scope decisions

Defining the MVP was a significant part of the work. Several features — dispute resolution tools, history views, edge case handling — were identified early but deliberately left for v2. Getting the core transfer flow right and ensuring adoption was the priority for launch.

MVP scope map — what's in, what's v2

Outcome

The tool launched 3 months after kickoff across all place-type locations. There was initial resistance — operators weren't used to a digital process — but adoption grew steadily.

Results
  • 70% positive adoption measured through on-site surveys
  • 1,000+ operators onboarded across the network
  • Paper forms eliminated — chain of custody now fully digital

Mercado Libre Dashboard Consolidation

Monitor
Last Mile

Consolidating 2–3 internal tools and spreadsheets into a single live monitor — reducing operational oversight time by 30–40%.

Role
Senior UX Designer
Stakeholders
Managers & floor leads
Context
Mercado Libre Logistics
Hero image — Monitor Last Mile

The problem

Last Mile leaders were managing operations across multiple internal tools and spreadsheets at the same time — literally several screens open at once. Building a picture of what was happening required manually cross-referencing data that was never in sync.

This project had demand from two directions: managers needed traceability and a reliable source of truth; floor leads needed something live they could act on during the shift. The challenge was designing for both without creating a tool that tried to do too much.

Consolidation strategy

We audited every tool in use and mapped which tasks were being performed in each. The goal wasn't to merge everything into one screen — it was to identify what could be unified without creating information overload, and what belonged in a separate context.

Tool audit — consolidation mapping

Progressive disclosure became the core pattern: the top level answers the most common questions at a glance, with drill-down available when something needs investigation.

Final design — unified monitor view

Outcome

The MVP replaced the multi-screen setup that leaders had been relying on. Managers got the traceability they needed; floor leads got a single live view to manage the shift.

Results
  • 30–40% reduction in operational oversight time
  • 2–3 tools and spreadsheets replaced by one view
  • Single source of truth for both managers and floor leads

Fury

Mercado Libre's internal developer platform — where all applications are built, deployed, and managed. I joined the team to lead the UX restructuring of a tool that had never had a dedicated design resource.

Role
Senior UX Designer
Status
Ongoing — 2024–present
Users
Developers + expanding to new roles
Hero image — Fury platform overview

The context

Fury had been running for years without a UX team. The result was what you'd expect: inconsistent patterns across sections, no use of Mercado Libre's Andes design system, navigation that had grown organically without a clear structure, and flows that worked — but only if you already knew how they worked.

The platform is also expanding beyond its original audience — from developers only to other internal roles like HR — which means the UX debt that was tolerated by power users needs to be addressed before it scales to a broader audience.

What we're working on

The work covers several parallel tracks: auditing and redesigning key sections (Traffics, Segments, Apps, a new Home), migrating UI components to Andes, and improving navigation consistency across the platform.

The starting point for each section is the same: map the current flow, evaluate it against Nielsen's heuristics and Andes patterns, identify improvement opportunities, and prioritize with the team.

Flow audit — current state mapping

How we work

One thing I've been doing differently on this project is using AI tooling to move faster and stay closer to engineering. I use Figma's MCP to generate design specs automatically, then push those files directly to GitHub for the dev team. It shortens the handoff loop considerably.

I also use Claude to structure UX audits — comparing current flows against the Andes component library and Nielsen's heuristics to surface specific, actionable improvement points. It's not about automating the design work; it's about making the analysis and documentation part faster so there's more time for the actual decisions.

Figma MCP → GitHub handoff workflow

Early results

The project is ongoing, but some flows have already shipped. The most significant result so far came from a redesign of the demo app creation flow.

Demo app flow — post launch
  • 62% reduction in demo app creations — from 452 to 172 in the same period
  • Indicates users are now creating production apps correctly instead of relying on demo workarounds
